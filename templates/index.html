Theoretical Appendix & Methodologies

01. Ordinary Least Squares (OLS)

The core computational engine of this terminal relies on the OLS estimator. The objective is to estimate the true population parameters by minimizing the sum of the squared residuals (SSR) between observed targets and the model's predictions.

Y = β0 + β1X1 + β2X2 + ... + ε

Where Y is the dependent vector, β represents the estimated coefficients, X are the predictor matrices, and ε is the stochastic error term.

02. The Gauss-Markov Assumptions

For the OLS estimator to be Best Linear Unbiased Estimator (BLUE), the data matrix must satisfy the Classical Linear Regression Model (CLRM) parameters:

  • > Linearity: The parameters (β) must be linear.
  • > Exogeneity: Expected value of errors is zero; predictors are independent of errors.
  • > Homoskedasticity: The variance of the residuals remains constant across all observations.
  • > No Autocorrelation: Residuals are not correlated with each other over time or space.

03. Automated Diagnostic Testing

The Study.Protocol module actively intercepts the OLS calculation to test for assumption violations using native statsmodels processing.

Breusch-Pagan Test

Detects Heteroskedasticity. If the variance of errors grows alongside the independent variable, standard errors become biased, rendering hypothesis tests invalid.

Breusch-Godfrey Test

Detects Serial Autocorrelation. Crucial for time-series matrices. Evaluates if past errors influence current errors, indicating a missing explanatory variable.

04. Inference & Topology

Post-diagnostic, the system renders final evaluations. R-Squared identifies the percentage of variance explained by the model, while Adjusted R-Squared penalizes the matrix for incorporating non-significant vectors.

The Visual.Matrix engine subsequently maps predicted values against actual targets in a 2D topological space to visually identify outliers and model tracking efficiency.
